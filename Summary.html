LLM Speedrun Submission Summary

This document summarizes the development run logs, engineering choices, and final architecture implemented for the 2,000-step LLM optimization task.

1. Final Model Architecture

  • Parameters: 1,304,320 PASSED CAP (< 2M)
  • Layers (n_layer): 4
  • Heads (n_head): 8
  • Embedding Size (n_embd): 160
  • Block Context Size: 256
  • Weight Tying: Enabled (tie_weights = True)
  • Attention Projections: Bias-free (bias = False)

2. Key Optimization Improvements

Major Enhancements:
  • Extended History Window: Doubled block size context length to 256, allowing broader textual dependencies to stay within the attention matrix view.
  • Scaled Initialization: Modified initialization standard deviation to explicitly scale down by a factor of 1/√(2 × layers) to stabilize backpropagation across layers.
  • Regularization Strategy: Introduced fine-grained dropout (0.001) alongside weight-decay (0.01) regularized through AdamW execution steps.
  • Batch and Gradient Scale: Scaled input sequence batches to 32 and incorporated explicit L2 gradient norm clipping at 1.0.

3. Collaboration & Human-Machine Split

The engineering process was a collaborative effort between the developer and a coding agent:

  • Machine Contribution (65%): Drafted custom tensor-dimension configurations, implemented mathematical scaling limits, generated semantic documentation structures, and tracked internal boundary conditions.
  • Human Contribution (35%): Evaluated architecture parameters, identified performance friction points in deep networks, conducted hardware execution procedures, and configured local repo file deployment.
